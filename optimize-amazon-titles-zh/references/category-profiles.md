# 类目画像

根据商品身份选择最接近的画像。商品适用多个画像时，合并必要检查并采用更严格的风险等级。

## 通用消费品

适用：家居、厨房、办公、园艺、宠物、运动、旅行和基础工具。

建议顺序：

`品牌 + 核心产品类型 + 主要差异点 + 尺寸/数量/颜色`

保留：

- 产品类型
- 会影响购买决策的容量、尺寸、数量或变体
- 一个已经验证的材料或功能差异点

优先迁移：

- 过长的使用场景列表
- 次要材料和功能
- 礼品语言和主观质量声明

## 服装、鞋靴和配饰

建议顺序：

`品牌 + 产品类型 + 必要受众/性别 + 款式/材料 + 子体尺寸/颜色`

必要检查：

- 区分父体标题和子体变体标题。
- 根据目标 Schema 确认尺寸、颜色是否由系统追加或需要写入 `item_name`。
- 不得把某个子体的颜色或尺寸写入父体标题。
- 当包装数量和版型会显著区分商品时予以保留。

风险：中。不同 Product Type 和站点的变体展示方式可能不同。

## 电子和电气产品

建议顺序：

`品牌 + 产品类型 + 型号/接口 + 功率/容量 + 端口/包装数量`

保留：

- 型号或代际
- 连接器或接口
- 防止买错所需的功率、电压、容量或协议
- 必要的数量，以及线缆或配件是否包含

标记：

- 无证据的快充、防水、安全或认证声明
- 含糊的兼容列表
- 没有明确关系措辞的商标设备名称

## 汽车、替换件和兼容产品

建议顺序：

`品牌 + 零件类型 + 位置/关键规格 + Compatible with/Fits + 品牌/车型/年份`

保留：

- 零件类型和安装位置
- `Compatible with`、`Fits` 或同等关系措辞
- 防止装配错误所需的品牌、车型、年份、发动机、OE 号或尺寸

不得以改变覆盖范围的方式压缩年份或适配列表。如果 75 个字符无法容纳足够的适配信息，在标题中保留最高层级的安全标识，把已验证细节迁移到 Item Highlights 或适配属性，并标记 `MANUAL_REVIEW`。

风险：高。重点关注商标措辞、适配准确性和退货风险。

## 工业、五金和 B2B 零部件

建议顺序：

`品牌 + 部件类型 + 标准/材料 + 关键尺寸 + 包装数量`

保留：

- 标准、螺纹、线规、公差、材料牌号、尺寸或连接类型
- 包装数量
- 采购所需的型号或零件号

不得用宽泛利益点替代精确规格。当标题在限制内无法准确识别部件时标记 `MANUAL_REVIEW`。

## 美妆和个护

建议顺序：

`品牌 + 产品类型 + 形态/主要成分 + 净含量/数量 + 变体`

保留：

- 产品形态
- 净含量
- 防止买错所需的色号、香型、肤质/发质或变体

治疗、临床、低敏、有机、无毒、无动物实验和认证声明均需证据。不得把化妆品利益点改写为医疗声明。

风险：高。声明和站点限制需要人工关注。

## 食品、补充剂和消耗品

建议顺序：

`品牌 + 产品类型 + 口味/形态 + 合法强度 + 净含量/数量`

保留：

- 口味或形态
- 数量、重量、体积和适用浓度
- 已验证的饮食属性

疾病、治疗、预防、减重、免疫、FDA 批准、有机、无过敏原和性能声明均需标记。不得推断营养或成分属性。

风险：高。声明或法规分类不清时默认 `MANUAL_REVIEW`。

## 婴幼儿和儿童产品

建议顺序：

`品牌 + 产品类型 + 已验证年龄/体重范围 + 关键尺寸/数量/使用边界`

保留：

- 相关且准确的年龄或体重范围
- 产品类型和预期用途
- 尺寸、数量和安全关键限制

睡眠、医疗、发育、安全、无毒、BPA-free 和认证声明均需证据。确认标题、包装、说明书、测试和 Listing 使用场景一致。

风险：高。安全敏感商品默认 `MANUAL_REVIEW`。

## 玩具和游戏

建议顺序：

`品牌 + 产品类型/游戏名称 + 已验证年龄范围 + 件数/人数 + 变体`

保留：

- 年龄分级
- 件数或玩家数量
- 仅在获得授权时使用角色或 IP 授权措辞

缺少证据时标记发育、教育、安全和授权声明。

## 套装、多包装和组合

建议顺序：

`品牌 + 套装身份 + 主要组成 + 总数量/尺寸`

保留：

- 准确的包含组件
- 包装或件数
- 尺寸和变体边界

不得暗示未包含的配件。对照结构化包装数据和图片审核标题内容。虚拟套装或混合品牌套装标记 `MANUAL_REVIEW`。

## 媒介类目

在最近一次政策核验时，图书和 DVD 不受 2026 年 75 字符规则影响。不要仅因标题较长就强制压缩。核对当前媒介规则，并按要求保留作品标准名称、版本、作者或艺人、载体格式和卷册信息。
