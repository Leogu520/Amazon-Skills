# 风险与声明审核

只要标题修改可能改变事实、法律关系、兼容关系、变体或安全含义，就标记 `MANUAL_REVIEW`。

## 证据敏感声明

包括但不限于：

- BPA-free、无毒、食品安全、儿童安全
- 防水或耐水等级
- 有机、天然、可持续、环保、可回收
- 低敏、皮肤科测试、临床验证
- 医疗级、FDA 批准、预防、治疗、治愈、缓解
- 防火、阻燃、抗菌
- 已认证、合规、已测试、已批准
- 精确的时长、速度、强度、负载、温度或效率

必须有明确的证据边界，例如测试报告、认证范围、包装声明、受控产品规格或用户明确确认。不得从竞品 Listing 推断证据。

调用内置校验器时，必须提供精确的规范化声明 ID 或精确别名。支持的规范化 ID 为：

`bpa_free`、`non_toxic`、`food_safe`、`child_safe`、`waterproof`、`organic`、`hypoallergenic`、`clinical`、`medical_approval`、`environmental`、`antimicrobial`、`fire_performance`、`age_life_stage`、`certification_compliance`、`educational_positioning` 和 `performance_duration`。

校验器会规范化大小写、空格和标点，但不会做子字符串匹配。`claim` 之类的泛化值不能验证 `waterproof`。

## 兼容关系和商标措辞

- 当第三方产品与被引用品牌的自有产品需要区分时，保留 `Compatible with`、`Fits`、`Replacement for` 或同等关系短语。
- 不得把第三方兼容标题缩短成可能暗示来源、赞助或品牌所有权的措辞。
- 根据结构化来源核对品牌、型号、代际、年份、发动机、接口和 OE/零件号。
- AI 改写引入任何未出现在已验证来源中的商标时必须标记。

## 父子变体

- 不得把某个子体的颜色、尺寸、口味、数量或型号复制到父体标题。
- 核对目标站点是从 `item_name` 展示变体属性，还是另行追加。
- 审核最终展示的子体标题，而不仅是源父体标题。
- 下游系统可能追加变体文本时预留字符空间。

## 套装和包装内容

- 保留准确数量和包含组件。
- 不得把 `works with` 改成 `includes`。
- 没有结构化证据时，不得暗示包含线缆、电池、保护壳、支架、补充装或配件。

## 合规和安全敏感产品

以下产品默认人工审核：

- 婴儿睡眠、约束、背带、喂养和耐用型婴儿用品
- 医疗器械和提出健康声明的产品
- 补充剂和可摄入产品
- 使用治疗性语言的化妆品
- 防护装备和具有安全等级的产品
- 电池、充电器、电气安全和无线电产品

标题可能影响产品分类。短标题不得删除准确分类所需的预期用途，也不得引入产品无法支持的新用途。

## 共享详情页和数据归属

- 多个贡献者提供目录数据时，卖家提交的标题未必控制实时详情页。
- 修改前记录源标题和拟修改标题。
- 获批修改后检查 `View Change History`。
- 识别可能相互覆盖的 ERP、PIM、Feed、模板、API 和手工编辑来源。

## 多语言 Listing

- 每个站点重新确定字符优先级，不要逐字翻译英文标题。
- 保留当地标准单位、数字格式和产品术语。
- 同时统计 Unicode Code Point 和 UTF-8 字节。
- 机器翻译的兼容、安全、年龄和法规术语必须人工审核。
- 当前确定性语义模式以英文为主。其他标题语言保留长度和硬性字符结果，同时增加 `UNSUPPORTED_TITLE_LANGUAGE` 并使用 `MANUAL_REVIEW`。
